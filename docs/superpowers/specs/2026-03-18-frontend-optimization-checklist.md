# 前端界面优化实施清单（IA 优先）

## 1) P0（先做，影响最大）

- `src/App.vue`：重构全局信息架构（IA）入口层级，首屏仅保留 1 个主行动按钮；补齐全局返回路径与固定底部主导航，确保关键路径不超过 4 步。
- `src/App.vue`：统一页面容器与安全区（顶部/底部）处理，避免移动端内容被状态栏或底部手势区遮挡。
- `src/views/PhotoInput.vue`：将拍照/选图/识别结果改为单列主流程，先展示“下一步动作”，再展示辅助信息；减少并列操作按钮。
- `src/views/PhotoInput.vue`：为 OCR 失败增加就地可恢复动作（重试、重新拍照、改为文本输入），保留用户已输入/已选内容。
- `src/views/TextInput.vue`：将“输入 -> 分析 -> 保存/继续”压缩为线性流程，默认聚焦输入区并突出主 CTA。
- `src/views/TextInput.vue`：优化软键盘弹起时布局（按钮吸底或跟随输入区），确保主 CTA 始终可见可点。

## 2) P1（次优先，提升连续体验）

- `src/views/Collection.vue`：优化列表信息密度与分组（最近、待复习等），优先展示下一步可执行操作，降低回看与筛选成本。
- `src/views/Collection.vue`：补齐空状态与异常状态的引导动作（去录入、重试加载、清空筛选）。
- `src/views/SentenceDetail.vue`：重排详情层级（原句、核心分析、例句/扩展），首屏聚焦“继续学习”相关按钮。
- `src/views/SentenceDetail.vue`：提供可恢复编辑与撤销路径，避免误操作后只能返回列表重来。
- `src/stores/index.js`（或相关 store）：沉淀跨页流程状态（最近入口、未完成步骤、错误上下文），支持中断后续接。
- `src/stores/index.js`（或相关 store）：统一 loading/error/retry 状态模型，给页面提供一致的可恢复交互能力。

## 3) P2（视觉系统与一致性收口）

- `src/assets/styles.css`：建立移动端优先的设计 token（间距、字号、圆角、阴影、层级、触控尺寸），统一按钮最小高度与点击热区。
- `src/assets/styles.css`：补齐通用状态样式（默认/按下/禁用/错误/成功）与表单反馈样式，减少页面内重复定义。
- `src/components/*`：梳理并统一基础组件接口（Button、Input、Card、Toast/Alert），收敛变体命名与默认行为。
- `src/components/*`：为关键组件补齐无障碍属性（语义标签、focus 可见性、aria 提示）与移动端触控反馈。
- `src/api/ocr.js` 与 `src/api/analyze.js`：标准化错误码与错误文案映射，区分网络失败、配额限制、解析失败等场景。
- `src/api/ocr.js` 与 `src/api/analyze.js`：返回结构增加可恢复建议字段（retryable、recommendedAction），前端可直接渲染恢复按钮。

## 4) 验收脚本（手工）

- [ ] 在 390x844 移动视口下，从首页可“一次点击”进入主输入路径（拍照或文本其一）。
- [ ] 核心路径点击步数达标：首次录入 <= 4 步，重复录入 <= 3 步。
- [ ] OCR/分析失败时，界面提供可恢复动作（重试、切换输入方式、保留已填内容）。
- [ ] 软键盘弹起后，输入区与主 CTA 不被遮挡，且无需额外滚动即可完成提交。
- [ ] 主要触控目标（按钮、列表项、可点图标）可点击区域 >= 44px。
