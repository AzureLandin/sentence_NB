# 2026-03-18 移动端 UX 导航设计（Approach A：信息架构优先）

## 目标与范围

- 目标：以信息架构（IA）先行重构移动端导航与关键流程，降低学习成本、缩短到达核心功能路径、提升录入与复习完成率。
- 范围：照片录入、文本录入、收藏列表、句子详情、全局导航与状态反馈；不包含 AI 分析模型、后端接口协议、桌面端专属交互重做。
- 设计原则：先明确页面层级与任务流，再做视觉和交互细化；所有优化以单手操作和小屏可读性为前提。

## 全局框架优化

- 采用 3 层 IA：`主导航层（入口） -> 任务页层（录入/浏览） -> 内容层（详情/编辑）`，避免跨层跳转。
- 底部主导航固定 3-4 项：`拍照录入`、`文本录入`、`收藏`、`设置`（可选）；当前页高亮且可感知返回路径。
- 页面头部统一：左侧返回（次级页）、中间标题、右侧上下文操作（如筛选/排序/更多），减少手势猜测成本。
- 路由状态可恢复：从详情返回时保留列表滚动位置、筛选条件与搜索词。
- 全局操作一致性：新增、保存、删除、重试的位置和文案保持一致，降低迁移成本。

## 视觉/可读性系统

- 建立移动端设计 Token：间距（4/8/12/16/24）、圆角（8/12）、阴影层级（2 档）、触控高度（最小 44px）。
- 文本层级固定：标题 18/20、正文 15/22、辅助 13/18；保证中英文混排在 320px 宽度下不拥挤。
- 强化信息对比：正文与背景对比度 >= 4.5:1，次要信息 >= 3:1；禁用低对比浅灰正文。
- 长内容策略：句子与分析结果默认折叠 3-5 行，提供“展开/收起”，防止首屏被单条内容占满。
- 组件反馈明确：按钮、输入、卡片提供 `默认/按下/禁用/错误` 四态，状态切换动画 <= 200ms。

## 照片录入页优化

- 首屏聚焦单任务：相机入口主按钮置底部拇指热区，辅入口（相册导入）次级展示。
- 拍照后流程改为线性：`拍照 -> OCR 处理中 -> 结果确认 -> 一键保存`，每步只暴露当前必要操作。
- OCR 结果按可编辑块展示：原文、分句建议、纠错建议分区；支持逐段快速确认而非整页编辑。
- 失败恢复路径明确：识别失败时提供 `重试识别`、`重新拍摄`、`改为手动输入` 三选项。
- 性能感知优化：处理超过 400ms 显示骨架与进度文案，超过 8s 自动提示网络/服务异常并给可执行操作。

## 文本录入页优化

- 输入区默认聚焦文本任务：顶部简短指引 + 多行输入框 + 主按钮“分析并保存”。
- 提供轻量快捷操作：粘贴清洗（去多余空格/换行）、一键分句、清空确认，均在输入框附近。
- 校验前置：空文本、超长文本、本地重复句在提交前即时提示，减少无效请求。
- 分析结果与编辑同屏分区：上方原文可改，下方结构化结果（语法/词汇/释义）可折叠查看。
- 提交后支持连续录入：保存成功停留当前页并清空输入，保留最近一次结果入口，提升批量录入效率。

## 收藏/详情优化

- 收藏页默认展示“最近学习”排序，提供常用筛选（标签、时间、难度）与关键字搜索。
- 列表卡片信息最小闭环：句子预览、更新时间、标签、复习状态，避免必须进详情才能判断价值。
- 详情页信息分块：原句、翻译、分析、笔记、复习记录；块级可折叠并记忆用户展开偏好。
- 详情操作底部固定：`编辑`、`复习`、`删除`（危险操作二次确认），确保单手可达。
- 列表与详情联动：详情修改后返回列表即时更新，不触发整页刷新与跳顶。

## 反馈与错误处理策略

- 反馈分级：`即时提示（Toast）`、`上下文错误（表单内）`、`阻断错误（全屏状态）`，避免一刀切弹窗。
- 文案规范：先说明问题，再给下一步动作，如“识别失败，请重试或改为手动输入”。
- 关键异步操作必须可见：提交、识别、保存均显示进行中状态，按钮禁用并防重复点击。
- 离线与弱网策略：请求失败自动重试 1 次；仍失败时保留本地草稿并提示稍后继续。
- 数据安全兜底：编辑中离开页面触发未保存提醒；自动草稿恢复仅在本机生效。

## 移动端人体工学

- 拇指热区优先：高频主操作放底部 1/3 区域，低频操作放顶部或二级菜单。
- 手势最小化：避免依赖隐藏手势完成关键任务，所有核心动作提供显式按钮。
- 输入体验：键盘弹起时主按钮跟随可见，不遮挡当前输入上下文。
- 安全边距：适配刘海屏/底部 Home 指示条，关键按钮与边缘保持 >= 12px。
- 中断友好：来电/切后台后恢复到最近步骤，不丢失输入与临时识别结果。

## 量化验收标准

- 导航效率：新用户在 30 秒内到达任一核心入口（拍照/文本/收藏）的成功率 >= 95%。
- 任务时长：文本录入“输入到保存”中位时长 <= 25 秒；照片录入“拍照到保存”中位时长 <= 45 秒。
- 交互成本：核心流程（拍照录入、文本录入、收藏检索）主任务点击步数分别 <= 4、<= 3、<= 3。
- 可读性：移动端关键页面正文字号不低于 15px，关键文本对比度满足 WCAG AA。
- 稳定性：OCR/分析请求失败后可恢复完成率 >= 90%，重复提交率 <= 2%。
- 满意度：移动端导航相关反馈（易用性问卷）均分 >= 4.3/5，且“找不到入口”问题占比 < 5%。
